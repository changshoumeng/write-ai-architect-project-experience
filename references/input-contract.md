# 输入契约

本文件用于判断用户输入是否足够支撑 `write-ai-architect-project-experience` 继续执行。该 skill 面向中文用户，最小输入是“项目详细材料 + 目标岗位 JD”。

## 最小输入

| 输入 | 作用 | 缺失时动作 |
|---|---|---|
| 项目详细材料 | 提供项目事实、职责、架构、技术、交付与结果 | 必须追问，不得写最终稿 |
| 目标岗位 JD | 提供岗位能力要求、关键词、评价标准和取舍方向 | 必须追问，不得写最终稿 |

项目材料可以是项目经历详细稿、项目复盘、PRD、架构文档、代码路径、测试记录、交付记录等。JD 可以是岗位原文、截图转文字、招聘链接摘录或用户整理的职责/要求。

## 推荐补充输入

- 用户个人真实职责边界：主导、负责模块、协作、交付负责人、架构师、核心开发等。
- 目标公司/行业方向：ToB、制造业、工业 AI、AI Agent、RAG、平台架构等。
- 可公开边界：客户名称、业务指标、模型规格、数据量、上线范围是否可写。
- 证据材料：代码路径、测试记录、评测结果、上线/验收/交付记录、客户反馈。
- 输出约束：字数、简历风格、是否偏业务/技术/管理/交付。

## JD 能力抽取

从 JD 中抽取：

- **Skills**：技术栈、工具、框架、工程方法，如 LangGraph、RAG、FastAPI、Vue、Docker、CI/CD。
- **Knowledge**：AI Agent、LLM、RAG、知识图谱、模型评测、系统架构、业务领域知识。
- **Abilities**：业务抽象、架构设计、复杂问题拆解、技术选型、跨团队协作、排障、交付。
- **Other Characteristics**：owner 意识、沟通影响力、风险意识、学习能力、客户导向、结果导向。

抽取后形成 5-10 个关键能力项，并按 JD 重要程度排序。

## 项目事实抽取

写作前至少抽取：

- 项目名称、项目时间、项目角色。
- 目标用户、业务场景、核心痛点、项目目标。
- 个人职责：业务侧、架构侧、实现侧、交付侧。
- 核心架构：前端、后端、AI Workflow、数据层、集成点。
- AI 设计：Agent 角色、编排、工具、路由、重试、Trace、Guardrails。
- 知识工程：结构化/非结构化知识、知识图谱、向量检索、RAG、知识抽取。
- 模型工程：模型、微调、Prompt、Few-Shot、CoT、评测集、专家评分。
- 工程交付：测试、部署、可观测性、客户验证、上线或交付边界。
- 高风险表述：数字、生产状态、客户名称、团队规模、ROI、独立主导。

## 项目事实到 JD 的映射

对每个 JD 关键能力项标记：

| 标记 | 含义 | 写法 |
|---|---|---|
| 强匹配 | 项目事实能直接证明该能力 | 放入主版本或核心技术 |
| 弱匹配 | 项目材料提到，但证据不足 | 保守表达，并在最终自检说明 |
| 缺口 | JD 重要，但项目材料无事实支撑 | 追问用户补充；无补充则不写 |
| 不可写 | 无证据、涉密或明显夸大 | 删除或匿名化 |

## 证据强度分级

| 级别 | 证据 | 可用表述 |
|---|---|---|
| L3 强证据 | 代码、架构文档、测试、交付记录直接支撑 | `设计并落地`、`实现`、`验证`、`交付` |
| L2 叙述证据 | 项目材料中明确写到，但缺少独立佐证 | 可写 `负责/主导`，高风险内容需在最终自检说明 |
| L1 弱证据 | 只能从上下文推断 | `形成方案`、`完成 PoC`、`参与规划`，或不写 |
| L0 无证据 | 没有来源 | 不写 |

## 必须追问的情况

遇到以下情况，先问用户，不进入下一步：

- 没有目标岗位 JD。
- 没有项目详细材料。
- 项目角色或职责边界不清。
- JD 最关键能力在项目材料里没有对应事实。
- 用户想写量化指标，但未提供指标来源。
- 用户想写客户名称、模型规格、数据量、上线范围，但公开边界不清。

追问示例：

```markdown
为了让项目经历精准匹配该 JD，还需要补充：
1. 目标岗位 JD 原文或核心要求是什么？
2. 该项目中你个人真实负责哪些部分？
3. JD 中最重要的能力要求，项目里有哪些事实可支撑？
4. 是否有 PoC/MVP/上线/验收/评测/交付证据？
5. 客户名称、指标、模型规模、数据量是否可以写入简历？
```

## 保守表达库

- `完成 PoC/原型/MVP 验证`
- `打通关键业务闭环`
- `形成可复用方案`
- `沉淀知识工程/Agent 工作流能力`
- `支撑后续上线/推广/迭代`
- `建立治理基础`
- `在材料范围内可证明`
- `仍需补充验收记录/评测报告/代码路径进一步佐证`
